# Story 14.2: Relatórios Regulatórios — SNGPC, Anvisa, Receituário e Agendamento

## Status

Done

## Story

**As a** supervisor de armazém (WMS_SUPERVISOR),
**I want** gerar e visualizar relatórios regulatórios (SNGPC, Anvisa, Receituário) e configurar o agendamento automático,
**so that** a empresa mantenha conformidade regulatória com órgãos sanitários e farmacêuticos.

## Complexity

**Estimativa:** M (Medium) · ~3-4 dias

## Dependencies

- Story 9.1 ✅, 9.2 ✅
- Backend 7.1 Done ✅ — `/api/v1/reports`

## Out of Scope

- ❌ Geração de XML SNGPC offline
- ❌ Envio automático para Anvisa

## Acceptance Criteria

1. **Página `/reports`** — relatórios regulatórios:
   - Abas: SNGPC | Anvisa | Receituário | Agendamentos
   - Cada aba: filtros de período (from/to), botão "Gerar Relatório"
   - Tabela de relatórios gerados: ID, tipo, período, status (PENDING/GENERATED/ERROR), data geração
   - Botão "Download" para relatórios GENERATED

2. **Agendamentos** (aba Agendamentos):
   - Lista de agendamentos ativos (tipo, frequência, próxima execução)
   - Botão criar/desativar agendamento (ADMIN)

3. **React Query hooks** — `lib/api/reports.ts`:
   - `useReports(filters?)` — GET /reports?type=&status=
   - `useGenerateReport` — POST /reports mutation
   - `useSchedules()` — GET /reports/schedules
   - `useCreateSchedule` — POST mutation
   - `useToggleSchedule(id)` — PUT mutation

4. **TypeScript types** — `types/reports.ts`:
   - `ReportType`, `ReportStatus`, `Report`, `ReportSchedule`
   - `GenerateReportRequest`, `CreateScheduleRequest`

5. **Sidebar**: rota `/reports` → `implemented: true`

6. **Testes unitários** (≥ 6 novos)
