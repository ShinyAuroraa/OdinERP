# Story 4.2: Rastreabilidade: Lote, Série, Validade & GS1

## Status

InReview

## Story

**As a** supervisor de armazém ou auditor do sistema (WMS Supervisor / WMS Admin),
**I want** consultar o histórico completo de movimentações de qualquer item por lote, número de série ou data de validade — com parsing e geração de códigos GS1 (EAN-13, GS1-128, SSCC, QR Code) — e visualizar a árvore completa de rastreabilidade desde o recebimento até a expedição,
**so that** seja possível responder a recalls em horas (não dias), rastrear a cadeia de custódia de qualquer item, garantir conformidade regulatória (ANVISA, Receita Federal) e operar com FEFO (First Expired First Out) de forma confiável.

## Acceptance Criteria

### AC1 — GET /traceability/lot/{lotNumber} (histórico de movimentos por lote)
**Given** um usuário autenticado com role WMS e `lotNumber` pertencente ao tenant,
**When** chama `GET /traceability/lot/{lotNumber}`,
**Then** retorna `200 OK` com lista de `StockMovementResponse` ordenada por `createdAt` ASC:
```json
{
  "lotId": "uuid",
  "lotNumber": "LOT-2026-001",
  "productId": "uuid",
  "productCode": "SKU-001",

    {
      "id": "uuid",

      "quantity": 100,
      "sourceLocationId": null,
      "sourceLocationCode": null,
```

### AC4 — GET /traceability/product/{productId}/expiry (consulta por validade — FEFO)
**Given** um `productId` e parâmetro opcional `warehouseId` e `expiryBefore` (formato ISO date: `YYYY-MM-DD`),
**When** chama `GET /traceability/product/{productId}/expiry`,
**Then** retorna `200 OK` com lista de `ExpiryResponse` ordenada por `expiryDate` ASC (FEFO):
```json
[
  {
    "lotId": "uuid",
    "lotNumber": "LOT-2026-001",
    "expiryDate": "2026-03-31",
    "daysUntilExpiry": 37,
    "quantityAvailable": 50,
    "locationCode": "A-01-B-02",
    "warehouseId": "uuid"

```
**And** inclui apenas lotes com `quantityAvailable > 0` e `Lot.status = AVAILABLE`.
**And** se `expiryBefore` fornecido → filtra apenas lotes expirando antes dessa data.

### AC5 — POST /gs1/parse (parsing de código GS1)
**Given** um código de barras GS1 em formato texto,
**When** chama `POST /gs1/parse` com body:
```json
{ "barcode": "(01)07891234567890(10)LOT001(17)261231(21)SN001", "format": "GS1_128" }
```
**Then** retorna `200 OK` com `GS1ParsedResponse`:
```json
{
  "format": "GS1_128",
  "applicationIdentifiers": {
    "01": { "name": "GTIN", "value": "07891234567890" },
    "10": { "name": "BATCH_LOT", "value": "LOT001" },
    "17": { "name": "EXPIRY_DATE", "value": "2026-12-31" },
    "21": { "name": "SERIAL_NUMBER", "value": "SN001" }
  },
  "gtin": "07891234567890",
  "lotNumber": "LOT001",
  "serialNumber": "SN001",
  "expiryDate": "2026-12-31"
}
```
**And** suporte a formatos: `EAN13`, `GS1_128`, `SSCC`, `QR_CODE` (GS1 DataMatrix).
**And** `400 Bad Request` se formato inválido ou AI não reconhecido.
**And** EAN-13 validado com algoritmo de check digit.

### AC6 — GET /gs1/generate (geração de código GS1)
**Given** parâmetros: `gtin` (obrigatório, 14 dígitos), `lotNumber` (opcional), `serialNumber` (opcional), `expiryDate` (opcional, YYYY-MM-DD),
**When** chama `GET /gs1/generate?gtin=07891234567890&lotNumber=LOT001&expiryDate=2026-12-31`,
**Then** retorna `200 OK` com `GS1GeneratedResponse`:
```json
{
  "ean13": "7891234567890",
  "gs1128": "(01)07891234567890(10)LOT001(17)261231",
  "sscc": null,
  "humanReadable": "[GTIN] 07891234567890 [LOTE] LOT001 [VALIDADE] 26-12-31"
}
```
**And** SSCC gerado apenas quando `serialNumber` fornecido (prefixo configurável via `wms.gs1.company-prefix`).
**And** `400 Bad Request` se `gtin` não tem 14 dígitos ou check digit inválido.

### GS1 Application Identifiers — Implementação

Implementar `GS1Parser` com mapeamento de AIs sem dependência externa pesada:

```java
// AIs suportados (focar nesses na implementação):
Map<String, GS1AI> KNOWN_AIS = Map.of(
    "00", new GS1AI("SSCC", 18, false),
    "01", new GS1AI("GTIN", 14, false),
    "10", new GS1AI("BATCH_LOT", 20, true),   // variable length
    "11", new GS1AI("PRODUCTION_DATE", 6, false),
    "17", new GS1AI("EXPIRY_DATE", 6, false),
    "21", new GS1AI("SERIAL_NUMBER", 20, true) // variable length
);

// Format: "(AI)VALUE(AI)VALUE..." para GS1-128
// EAN-13: string de 13 dígitos com check digit (Luhn modificado)
// SSCC: "(00)" + 18 dígitos
```

> Referência spec: GS1 General Specifications v24 (seção 3.2 — Application Identifiers).
> Para o check digit de EAN-13, usar algoritmo padrão GS1 (peso alternado 1/3).

### Lot.expiryDate — Nullable

`Lot.expiryDate` pode ser `null` para produtos sem controle de validade (ex.: parafusos, embalagens).
A JPQL `NULLS LAST` em `findAvailableByProductFefo` já trata corretamente.

### Input Validation — Path Params

Path params `{lotNumber}` e `{serialNumber}` são strings livres — validar no controller:
- Max length: 50 caracteres
- Charset: alfanumérico + hífen + underscore (regex: `^[A-Za-z0-9\\-_]+$`)
- `400 Bad Request` se violado (via `@Valid` + `@Pattern` ou validação manual)

Verificar versão: Spring Boot 3.2.x → Spring Data Elasticsearch 5.2.x → ES 8.x client.

> **NÃO adicionar** `co.elastic.clients:elasticsearch-java` separado — já transitivo via starter.
