# Story 8.4: App Android WMS — Inventário & Transferências

## Status

InReview

## Story

**As a** operador de armazém,
**I want** um fluxo completo de inventário físico e transferências internas no aplicativo Android WMS, com listagem de sessões ativas por corredor/posição, contagem de itens via scanner GS1, alertas de divergência vs. sistema, dupla contagem obrigatória para divergências acima do threshold, e criação/confirmação de transferências internas com scan de origem e destino,
**so that** eu possa executar inventários cíclicos e gerais e movimentações internas de estoque diretamente do dispositivo móvel — com ou sem conexão de rede — garantindo precisão de contagem, rastreabilidade de divergências e sincronização automática ao recuperar conectividade.

## Complexity

**Estimativa:** M (Medium) · ~3-4 dias de desenvolvimento

## Risks

| Risco | Probabilidade | Impacto | Mitigação |
|-------|--------------|---------|-----------|
| Room `MIGRATION_3_4` incompleta causa crash fatal em upgrade de usuários com v3 instalada | Alta | Alto | DDL completo das 4 tabelas + 3 índices em Dev Notes; usar `CREATE TABLE IF NOT EXISTS`; validar migration em emulador API 26+ antes de release |
| Índice parcial SQLite `WHERE counted_qty IS NOT NULL` pode não ser suportado via Room | Média | Médio | Usar índice composto `(session_id, product_code)` sem cláusula WHERE como alternativa safe; filtro WHERE na query JPQL em vez do índice |
| Dupla contagem por mesmo operador viola integridade do processo de inventário | Média | Alto | `InventoryItem` deve armazenar `firstCounterId` localmente; `doubleCount()` valida que `counterId` da 2ª contagem ≠ `firstCounterId` da 1ª |
| `loadCountingListFromCache()` retornando lista vazia (padrão QA-8.3-001 herdado) degrada UX offline de inventário | Alta | Médio | `InventoryRepository.getCountingList()` deve propagar dados reais do Room até o ViewModel; fix explícito documentado na Tarefa 4 e nos testes U10–U11 |
| Transfer race condition: `createTransfer()` + `confirmTransfer()` offline enfileirados; backend retorna 409 ao sincronizar | Baixa | Médio | Tratar HTTP 409 como `SYNCED` (idempotência); documentado no `InventorySyncWorker` como caso especial |
| `InventorySyncWorker.NOTIFICATION_ID` conflitando com `PickingSyncWorker(2000)` e `ReceivingSyncWorker` se não atomizado | Alta | Baixo | `AtomicInteger(3000)` no companion object (diferente dos workers anteriores); fix QA-8.2-004 aplicado preventivamente |

## Acceptance Criteria

### AC1 — Lista de Sessões de Inventário Ativas

**Given** operador autenticado com role `WMS_OPERATOR` (ou superior),
**When** acessa a tela "Inventário" via menu de Operações,
**Then**:
- Lista exibe sessões ativas via `GET /api/v1/inventory/sessions?status=ACTIVE&tenantId={tenantId}`
- Cada card de sessão exibe: número da sessão, tipo (CYCLIC/FULL em português: "Cíclico"/"Geral"), corredor, progresso (ex: "3/10 itens contados"), status
- Pull-to-refresh atualiza a lista
- Em modo offline: lista vem do Room (`inventory_sessions_cache`) com banner "Dados offline — mostrando cache local"
- Estado vazio (sem sessões ativas): mensagem "Nenhuma sessão de inventário ativa" com ícone de prancheta
- Erro de rede: mensagem de erro com botão "Tentar novamente"
- Dados do Room exibidos corretamente quando offline (fix QA-8.3-001 aplicado ao InventoryRepository)

---

### AC2 — Lista de Contagem por Corredor/Posição

**Given** operador na lista de sessões de inventário,
**When** seleciona uma sessão,
**Then**:
- Tela exibe lista de itens a contar via `GET /api/v1/inventory/sessions/{id}/counting-list?aisle={aisle}`
- Lista ordenada por posição dentro do corredor
- Cada item exibe: código do produto, descrição, posição, quantidade sistema (`system_qty`), quantidade contada (exibe "—" quando `counted_qty` ainda for nulo)
- Itens com contagem divergente destacados com badge vermelho (quantidade contada ≠ quantidade sistema)
- Itens com status `NEEDS_REVIEW` exibem badge amarelo "Revisão necessária"
- Filtro por corredor disponível (picker no topo da tela)
- Dados cacheados em Room (`inventory_items_cache`) para uso offline
- Estado de loading e erro com retry

---

### AC3 — Contagem de Item via Scanner

**Given** operador na tela de lista de contagem com item no status `PENDING` ou `OFFLINE_COUNTED`,
**When** escaneia código de barras do produto na posição,
**Then**:
- `BarcodeAnalyzer` existente detecta o código sem modificação
- `Gs1Parser` extrai: GTIN-14 (AI 01), número de lote (AI 10), data de validade (AI 17), número de série (AI 21)
- Tela de contagem (`InventoryCountScreen`) exibe: produto identificado (nome, GTIN), lote extraído do scan, posição, `system_qty` como referência, campo editável de `counted_qty` (pré-preenchido com `system_qty`)
- Campos de lote, posição e quantidade editáveis como fallback manual
- Botão "Confirmar Contagem" chama `PUT /api/v1/inventory/sessions/{id}/items/{itemId}/count` com `{ productCode, countedQty, lotNumber, position }`
- Sucesso: item marcado como `COUNTED` na lista; feedback visual + háptico (100ms)
- Em offline: confirmação enfileirada em `inventory_sync_queue` com `operationType=COUNT_ITEM`; item marcado localmente como `OFFLINE_COUNTED`
- Item retorna para `InventoryCountingListScreen` após confirmação

---

### AC4 — Alerta de Divergência

**Given** operador na `InventoryCountScreen` com quantidade contada preenchida,
**When** a `countedQty` digitada é diferente da `systemQty` do item,
**Then**:
- Divergência calculada em tempo real antes da confirmação: quantidade absoluta e percentual (ex: "+3 unid (15%)")
- Badge de divergência exibida abaixo do campo `countedQty`: "+{diff} unid ({pct}%)" em vermelho para positivo, "-{diff} unid ({pct}%)" para negativo
- Se divergência > threshold (configurável, default 10%): Snackbar de aviso com cor vermelha: "Divergência elevada: {pct}%. Confirme para registrar ou use Dupla Contagem."
- Aviso é **não bloqueante** — botão "Confirmar Contagem" permanece habilitado
- Botão "Dupla Contagem" visível apenas quando divergência > threshold
- Se divergência ≤ threshold: sem aviso especial; badge de divergência exibida em laranja (informativo apenas)
- Se `countedQty == systemQty`: badge não exibida; confirmação prossegue normalmente

---

### AC5 — Dupla Contagem (Double Count)

**Given** operador na `InventoryCountScreen` com divergência acima do threshold,
**When** pressiona o botão "Dupla Contagem",
**Then**:
- Navegação para `InventoryDoubleCountScreen`
- Tela exibe no header: resultado da 1ª contagem (operador, quantidade, timestamp)
- Campo de quantidade para a 2ª contagem (em branco, obrigatório)
- Botão "Confirmar 2ª Contagem" chama `POST /api/v1/inventory/sessions/{id}/double-count/{itemId}` com `{ countedQty, counterId }` onde `counterId` vem do JWT do operador atual
- Validação local: se `counterId` da 2ª contagem == `firstCounterId` armazenado no `InventoryItemCacheEntity`, exibir erro "Dupla contagem deve ser realizada por operador diferente"
- Se 2ª contagem confirma a 1ª (diferença ≤ 2 unidades): item marcado como `COUNTED_VERIFIED`; tela de sucesso
- Se 2ª contagem diverge da 1ª: item marcado como `NEEDS_REVIEW`; mensagem "Item requer revisão do supervisor"
- Em offline: operação enfileirada em `inventory_sync_queue` com `operationType=DOUBLE_COUNT`; status local atualizado otimisticamente

---

### AC6 — Submissão de Inventário

**Given** operador na `InventoryCountingListScreen` com todos os itens do corredor no status `COUNTED`, `COUNTED_VERIFIED` ou `NEEDS_REVIEW` (nenhum `PENDING`),
**When** pressiona o botão "Submeter Inventário",
**Then**:
- Botão "Submeter Inventário" habilitado somente quando nenhum item estiver com status `PENDING` ou `OFFLINE_COUNTED`
- `InventorySubmitScreen` exibe resumo da sessão: total de itens, itens sem divergência, itens com divergência aceita (`COUNTED_VERIFIED`), itens pendentes de revisão (`NEEDS_REVIEW`)
- Botão "Confirmar Submissão" chama `POST /api/v1/inventory/sessions/{id}/submit`
- Resposta de sucesso: tela de sucesso "Inventário submetido! Sessão #{numero}" com resumo de divergências registradas
- Botão "Voltar para lista" retorna para `InventorySessionListScreen`
- Em offline: operação enfileirada em `inventory_sync_queue` com `operationType=SUBMIT_SESSION`; tela de sucesso com aviso "Submissão pendente de sincronização"

---

### AC7 — Lista e Execução de Transferências Internas

**Given** operador autenticado com role `WMS_OPERATOR` (ou superior),
**When** acessa a tela "Transferências" via menu de Operações,
**Then**:
- Lista exibe transferências `PENDING` via `GET /api/v1/transfers?status=PENDING&tenantId={tenantId}`
- Cada card exibe: posição de origem, posição de destino, produto, quantidade, lote (se disponível), status
- Pull-to-refresh; banner offline; estado vazio "Nenhuma transferência pendente"; estado de erro com retry
- Dados cacheados em Room (`transfers_cache`) para uso offline
- Botão FAB "+" abre `TransferCreateScreen` para criação de nova transferência
- Fluxo de criação: scan da posição de origem (`sourceLocation`) → scan do produto (`productCode`) → campo de quantidade (`qty`) → scan da posição de destino (`destinationLocation`)
- Confirmação de criação chama `POST /api/v1/transfers` com `{ sourceLocation, destinationLocation, productCode, qty, lotNumber }`
- Após criação, `PUT /api/v1/transfers/{id}/confirm` confirma a transferência
- Em offline: operações enfileiradas em `inventory_sync_queue` com `operationType=TRANSFER`; status local `PENDING`
- Feedback visual + háptico ao confirmar

---

### AC8 — InventorySyncWorker (Sincronização Offline)

**Given** dispositivo sem conectividade com operações de inventário ou transferências enfileiradas,
**When** conectividade é restaurada,
**Then**:
- `InventorySyncWorker` (`@HiltWorker` + `@AssistedInject`, extends `CoroutineWorker`) ativado por `Constraints.Builder().setRequiredNetworkType(NetworkType.CONNECTED)`
- Worker processa `inventory_sync_queue` em ordem FIFO por `created_at`
- Para cada item com status `PENDING`: executa chamada REST correspondente por `operationType`:
  - `COUNT_ITEM` → `PUT /api/v1/inventory/sessions/{sessionId}/items/{itemId}/count`
  - `DOUBLE_COUNT` → `POST /api/v1/inventory/sessions/{sessionId}/double-count/{itemId}`
  - `SUBMIT_SESSION` → `POST /api/v1/inventory/sessions/{sessionId}/submit`
  - `TRANSFER` → `POST /api/v1/transfers` + `PUT /api/v1/transfers/{id}/confirm`
- Sucesso (2xx): item da fila marcado como `SYNCED`; Room atualizado com status final
- Conflito (409): item marcado como `SYNCED` (idempotência — transfer já existe remotamente)
- Conflito permanente (404): item marcado como `SYNC_CONFLICT`; notificação local via `NotificationCompat`
- Falha transitória (5xx / timeout): retry com incremento de `retry_count`; após 3 tentativas: `SYNC_FAILED` + notificação
- `NOTIFICATION_ID = AtomicInteger(3000)` no companion object (diferente de `PickingSyncWorker(2000)` e workers anteriores — fix QA-8.2-004 padrão)
- Worker registrado com tag `"inventory_sync"` (diferente de `"receiving_sync"`, `"picking_sync"`, `"general_sync"`)
- **FIX QA-8.3-001:** `InventoryRepository.loadCountingListFromCache()` retorna dados reais do Room (`inventoryItemDao.getBySessionId(sessionId)`); `InventoryViewModel` emite `CountingListLoaded(items, isOffline=true)` quando offline

```kotlin
// WmsDatabase.kt — adicionar:
val MIGRATION_3_4 = object : Migration(3, 4) {
    override fun migrate(database: SupportSQLiteDatabase) {
        database.execSQL("""
            CREATE TABLE IF NOT EXISTS inventory_sessions_cache (
                id TEXT NOT NULL PRIMARY KEY,
                tenant_id TEXT NOT NULL,
                session_number TEXT NOT NULL,
                session_type TEXT NOT NULL,
                status TEXT NOT NULL,
                aisle TEXT,
                total_items INTEGER NOT NULL DEFAULT 0,
                counted_items INTEGER NOT NULL DEFAULT 0,
                last_sync_at INTEGER NOT NULL
            )
        """.trimIndent())
        database.execSQL("""
            CREATE TABLE IF NOT EXISTS inventory_items_cache (
                id TEXT NOT NULL PRIMARY KEY,
                session_id TEXT NOT NULL,
                tenant_id TEXT NOT NULL,
                product_code TEXT NOT NULL,
                gtin TEXT NOT NULL,
                description TEXT NOT NULL,
                position TEXT NOT NULL,
                system_qty INTEGER NOT NULL,
                counted_qty INTEGER,
                double_count_qty INTEGER,
                lot_number TEXT,
                first_counter_id TEXT,
                local_status TEXT NOT NULL DEFAULT 'PENDING'
            )
        """.trimIndent())
        // Índice para queries por sessão e posição (ordem de contagem)
        database.execSQL("""
            CREATE INDEX IF NOT EXISTS idx_inventory_items_session
            ON inventory_items_cache(session_id, position)
        """.trimIndent())
        // Índice composto para queries de divergência (sem WHERE parcial — compatibilidade SQLite/Room)
        database.execSQL("""
            CREATE INDEX IF NOT EXISTS idx_inventory_items_divergence
            ON inventory_items_cache(session_id, product_code)
        """.trimIndent())
        database.execSQL("""
            CREATE TABLE IF NOT EXISTS inventory_sync_queue (
                id INTEGER PRIMARY KEY AUTOINCREMENT,
                tenant_id TEXT NOT NULL,
                operation_type TEXT NOT NULL,
                session_id TEXT NOT NULL,
                item_id TEXT,
                payload TEXT NOT NULL,
                status TEXT NOT NULL DEFAULT 'PENDING',
                retry_count INTEGER NOT NULL DEFAULT 0,
                created_at INTEGER NOT NULL
            )
        """.trimIndent())
        database.execSQL("""
            CREATE TABLE IF NOT EXISTS transfers_cache (
                id TEXT NOT NULL PRIMARY KEY,
                tenant_id TEXT NOT NULL,
                source_location TEXT NOT NULL,
                destination_location TEXT NOT NULL,
                product_code TEXT NOT NULL,
                qty INTEGER NOT NULL,
                lot_number TEXT,
                status TEXT NOT NULL,
                local_status TEXT NOT NULL DEFAULT 'PENDING',
                created_at INTEGER NOT NULL
```

### InventorySyncWorker — NOTIFICATION_ID AtomicInteger (Fix QA-8.2-004 padrão)

```kotlin
// InventorySyncWorker.kt
@HiltWorker
class InventorySyncWorker @AssistedInject constructor(
    @Assisted context: Context,
    @Assisted workerParams: WorkerParameters,
    private val inventorySyncQueueDao: InventorySyncQueueDao,
    private val inventoryItemDao: InventoryItemDao,
    private val transferDao: TransferDao,
    private val wmsApiService: WmsApiService
) : CoroutineWorker(context, workerParams) {

    companion object {
        const val TAG = "inventory_sync"
        // FIX QA-8.2-004: AtomicInteger para NOTIFICATION_ID — diferente de PickingSyncWorker(2000) e ReceivingSyncWorker
        private val NOTIFICATION_ID = AtomicInteger(3000)
    }

    override suspend fun doWork(): Result = withContext(Dispatchers.IO) {
        val tenantId = inputData.getString("tenantId") ?: return@withContext Result.failure()
        val pendingItems = inventorySyncQueueDao.getPendingByTenant(tenantId) // FIFO por createdAt

        for (item in pendingItems) {
            try {
                val response = when (item.operationType) {
                    "COUNT_ITEM" -> {
                        val req = json.decodeFromString<CountItemRequestDto>(item.payload)
                        wmsApiService.countItem(item.sessionId, item.itemId!!, req)
                    }
                    "DOUBLE_COUNT" -> {
                        val req = json.decodeFromString<DoubleCountRequestDto>(item.payload)
                        wmsApiService.doubleCountItem(item.sessionId, item.itemId!!, req)
                    }
                    "SUBMIT_SESSION" -> wmsApiService.submitInventorySession(item.sessionId)
                    "TRANSFER" -> {
                        val req = json.decodeFromString<CreateTransferRequestDto>(item.payload)
                        val createResponse = wmsApiService.createTransfer(req)
                        if (createResponse.isSuccessful) {
                            val transferId = createResponse.body()?.id ?: return@withContext Result.failure()
                            wmsApiService.confirmTransfer(transferId)
                        } else createResponse
                    }
                    else -> null
                }

                when {
                    response?.isSuccessful == true -> {
                        inventorySyncQueueDao.updateStatus(item.id, "SYNCED")
                    }
                    response?.code() == 409 -> {
                        // Idempotência: transferência já existe remotamente — tratar como SYNCED
                        inventorySyncQueueDao.updateStatus(item.id, "SYNCED")
                    }
                    response?.code() == 404 -> {
                        inventorySyncQueueDao.updateStatus(item.id, "SYNC_CONFLICT")
                        notifyUser("Conflito de sincronização", "Operação ${item.operationType} não encontrada remotamente.")
                    }
                    else -> handleRetry(item)
                }
            } catch (e: IOException) {
                handleRetry(item)
            }
        }
        Result.success()
    }

    private suspend fun handleRetry(item: InventorySyncQueueEntity) {
        if (item.retryCount >= 2) {
            inventorySyncQueueDao.updateStatus(item.id, "SYNC_FAILED")
            notifyUser("Falha na sincronização", "Operação ${item.operationType} falhou após 3 tentativas.")
        } else {
            inventorySyncQueueDao.incrementRetryCount(item.id)
        }
    }

    private fun notifyUser(title: String, message: String) {
        val notification = NotificationCompat.Builder(applicationContext, "inventory_sync_channel")
            .setSmallIcon(android.R.drawable.ic_dialog_alert)
            .setContentTitle(title)
            .setContentText(message)
            .setPriority(NotificationCompat.PRIORITY_DEFAULT)
            .build()
        NotificationManagerCompat.from(applicationContext)
            .notify(NOTIFICATION_ID.getAndIncrement(), notification)
    }
}
```
