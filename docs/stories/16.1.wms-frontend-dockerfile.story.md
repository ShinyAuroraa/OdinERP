# Story 16.1: Dockerfile Multi-Stage + Docker Compose (wms-web)

## Status

Done

## Story

**As a** engenheiro DevOps,
**I want** containerizar o `wms-web` com Dockerfile multi-stage otimizado,
**so that** possa fazer build e deploy consistente em qualquer ambiente (local, staging, produção).

## Complexity

**Estimativa:** M (Medium) · ~1 dia

## Dependencies

- EPIC-FE-001 ✅ (wms-web implementado e buildando)

## Out of Scope

- ❌ Push para registry (Story 16.3)
- ❌ K8s manifests (Story 16.4)
- ❌ CI/CD pipeline (Story 16.2)

## Acceptance Criteria

1. **`next.config.ts`** com `output: 'standalone'` para build otimizado Docker
2. **`apps/wms-web/Dockerfile`** multi-stage (deps → builder → runner):
   - Stage deps: instala dependências com `npm ci`
   - Stage builder: builda o app com variáveis ARG para env vars NEXT_PUBLIC_*
   - Stage runner: Alpine minimal, user não-root, imagem ≤ 300MB
3. **`apps/wms-web/.dockerignore`**: exclui `node_modules`, `.next`, `.git`, `*.test.*`, arquivos de dev
4. **`docker-compose.yml`** na raiz do projeto com 2 serviços:
   - `wms-web-dev`: modo desenvolvimento (volume mount + hot-reload)
   - `wms-web`: modo produção (imagem built)
5. **`docker build`** executa sem erros a partir de `apps/wms-web/`
6. **`npm run build`** continua passando após adição de `output: 'standalone'`
