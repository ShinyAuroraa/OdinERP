# Story 11.1: Recebimento de Mercadorias — Notas + Wizard de Conferência + Lotes

## Status

Done

## Story

**As a** operador de armazém (WMS_OPERATOR),
**I want** criar e gerenciar notas de recebimento de mercadorias via interface web — com wizard de conferência item por item, atribuição de lotes e números de série, aprovação de divergências por supervisor,
**so that** a entrada de mercadorias seja rastreada com precisão e os dados de lote/validade sejam capturados corretamente antes do putaway.

## Complexity

**Estimativa:** L (Large) · ~4-5 dias de desenvolvimento
[Wizard multi-step de conferência + status machine + aprovação de divergências por role]

## Dependencies

- Story 9.2 concluída ✅ — Design System completo

## Out of Scope

- ❌ Mobile scanner de código de barras no wizard

## Acceptance Criteria

1. **Página `/receiving`** — lista de notas de recebimento com DataTable:
   - Colunas: Nº Nota, Armazém, Fornecedor, Ref. OC, Status (StatusBadge), Data, Ações
   - Filtro por status (PENDING/IN_PROGRESS/COMPLETED/FLAGGED) + filtro por armazém
   - Botão "Nova Nota" (WMS_OPERATOR+)
   - Loading skeleton + empty state "Nenhuma nota de recebimento cadastrada"

2. **Formulário de Nova Nota** (modal):
   - Campos: armazém (select), doca de chegada (locationId), fornecedor, referência OC, itens
   - Adicionar itens: SKU/produto (select), quantidade esperada
   - Submit → `POST /api/v1/receiving-notes` → status PENDING → toast sucesso

3. **Wizard de Conferência** `/receiving/{id}`:
   - PageHeader com número da nota + status badge + breadcrumb
   - Botão "Iniciar Conferência" → `POST /receiving-notes/{id}/start` (status → IN_PROGRESS)
   - Lista de itens com status por item (PENDING/CONFIRMED/FLAGGED)
   - Para cada item: modal com campos: quantidade recebida, lote, fabricação, validade, código GS1, seriais (textarea)
   - Confirmar item → `POST /receiving-notes/{id}/items/{itemId}/confirm`
   - Botão "Finalizar" → `POST /receiving-notes/{id}/complete`
   - Se houver itens FLAGGED: botão "Aprovar Divergências" (apenas WMS_SUPERVISOR) → `POST /approve-divergences`

4. **Status Machine visual**:
   - Steps component: Criada → Conferência → Aprovação (se necessário) → Concluída
   - Status badges: PENDING (cinza), IN_PROGRESS (azul), COMPLETED (verde), FLAGGED (laranja)

5. **React Query hooks** — `lib/api/receiving.ts`:
   - `useReceivingNotes(filters?)` — GET /api/v1/receiving-notes, staleTime 30s (dados operacionais)
   - `useReceivingNote(id)` — GET /api/v1/receiving-notes/{id}
   - `useCreateReceivingNote` — POST /api/v1/receiving-notes
   - `useStartConference(id)` — POST /start
   - `useConfirmItem(noteId, itemId)` — POST /items/{id}/confirm
   - `useCompleteNote(id)` — POST /complete
   - `useApproveDivergences(id)` — POST /approve-divergences

6. **TypeScript types** — `types/receiving.ts`:
   - `ReceivingNote`, `ReceivingNoteItem`, `ReceivingNoteStatus`, `ReceivingItemStatus`
   - `CreateReceivingNoteRequest`, `ConfirmReceivingItemRequest`

7. **Proteção por role**:
   - Listagem + Conferência: WMS_OPERATOR+
   - Aprovar Divergências: apenas WMS_SUPERVISOR+
   - Botão "Aprovar Divergências" oculto para OPERATOR

8. **Testes unitários** (≥ 10 novos):
   - ReceivingList: render, loading, empty, filtro status
   - ReceivingForm: submit válido, erro validação
   - ReceivingDetail: botão iniciar, lista itens, status badges
   - Hooks: useReceivingNotes mock
