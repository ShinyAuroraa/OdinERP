# Story 2.1: Cadastro de Armazéns e Estrutura Física (CRUD)

## Status

Done

## Story

**As a** administrador de armazém (WMS Admin),
**I want** gerenciar a estrutura física completa do armazém via APIs REST — cadastrando armazéns, zonas, corredores, prateleiras e posições com validações de negócio e endereçamento hierárquico automático,
**so that** operadores possam referenciar localizações precisas no armazém (ex: `WH-001/ZN-01/A01/S02/L003`) e o sistema tenha a estrutura completa necessária para movimentações de estoque nas Waves 3+.

## Complexity

**Estimativa:** L (Large) · ~5-8 dias de desenvolvimento
[Fonte: EPIC-WMS-001 — Wave 2 é a maior wave de cadastros, 5 entidades hierárquicas]

## Out of Scope

> Itens explicitamente **não implementados** nesta story:

- ❌ Validação de capacidade vs. estoque existente (constraint de ocupação) → Wave 4
- ❌ Importação em lote de estrutura física (CSV/Excel) → Story de importações futuras
- ❌ Endereçamento automático gerado pelo sistema (sem input de código) → pode ser adicionado via enhancement
- ❌ Histórico de movimentações de localização (tipo rastreio de mudanças físicas) → Wave 6 auditoria avançada
- ❌ Reativação de entidades desativadas via endpoint → operação manual no DB (risco de integridade)
- ❌ Paginação nas listagens → stories de listagem avançada (aceitável para a wave 2, volumes ainda baixos)
- ❌ Filtros avançados além de `?type=` em `/locations` → story separada de search/filter

## Business Value

**Por que esta story é prioritária na Wave 2:**

- **Fundação da hierarquia WMS:** Toda movimentação de estoque (Waves 3-6) referencia `location_id`. Sem estrutura cadastrada, nenhuma operação de armazém é possível.
- **Endereçamento único por tenant:** Código único por tenant + `fullAddress` derivado da hierarquia garante rastreabilidade de qualquer item físico no armazém.
- **Multi-tenant isolado:** Cada operação usa `tenantId` do JWT → impossível acessar dados de outro cliente.
- **Tipos funcionais de localização:** `LocationType` (PICKING, RECEIVING_DOCK, QUARANTINE, etc.) permite que regras de negócio das waves 3+ filtrem localizações por função.

## Risks

| Risco | Probabilidade | Impacto | Mitigação |
|-------|--------------|---------|-----------|
| `fullAddress` com lazy loading N+1 ao criar Location | Média | Médio | `@Transactional` + carregamento explícito da cadeia Shelf→Aisle→Zone→Warehouse no serviço |
| Conflito de `code` único por tenant sem índice composto no serviço | Baixa | Alto | Repositories já têm `existsBy...` + `BusinessException` antes do INSERT |
| Tenant A acessar warehouse do Tenant B via UUID direto | Baixa | Alto | Todos os findById filtram por `tenantId` (validação explícita no service) |
| Controller integration tests com JWT mock complexo | Média | Médio | Usar `SecurityMockMvcRequestPostProcessors.jwt()` com `tenant_id` claim |
| `fullAddress NOT NULL` no DB sem valor → ConstraintViolation | Baixa | Alto | LocationService DEVE calcular `fullAddress` antes de salvar; coberto por teste |

## Acceptance Criteria

1. **Warehouse CRUD** — `WarehouseController` em `/api/v1/warehouses`:
   - `POST /api/v1/warehouses` → 201 Created com `WarehouseResponse` (código único por tenant, nome obrigatório)
   - `GET /api/v1/warehouses` → 200 OK com lista de armazéns do tenant
   - `GET /api/v1/warehouses/{id}` → 200 OK ou 404 Not Found (isolado por tenant)
   - `PUT /api/v1/warehouses/{id}` → 200 OK com dados atualizados
   - `PATCH /api/v1/warehouses/{id}/deactivate` → 204 No Content (seta `active=false`)

2. **Zone CRUD** — `ZoneController` aninhado sob warehouse:
   - `POST /api/v1/warehouses/{warehouseId}/zones` → 201 Created (código único por tenant+warehouse)
   - `GET /api/v1/warehouses/{warehouseId}/zones` → 200 OK com lista de zonas da warehouse
   - `GET /api/v1/zones/{id}` → 200 OK ou 404 (isolado por tenant)
   - `PUT /api/v1/zones/{id}` → 200 OK
   - `PATCH /api/v1/zones/{id}/deactivate` → 204 No Content

3. **Aisle CRUD** — `AisleController` aninhado sob zone:
   - `POST /api/v1/zones/{zoneId}/aisles` → 201 Created (código único por tenant+zone)
   - `GET /api/v1/zones/{zoneId}/aisles` → 200 OK
   - `GET /api/v1/aisles/{id}` → 200 OK ou 404 (isolado por tenant)
   - `PUT /api/v1/aisles/{id}` → 200 OK

4. **Shelf CRUD** — `ShelfController` aninhado sob aisle:
   - `POST /api/v1/aisles/{aisleId}/shelves` → 201 Created (código único por tenant+aisle)
   - `GET /api/v1/aisles/{aisleId}/shelves` → 200 OK
   - `GET /api/v1/shelves/{id}` → 200 OK ou 404 (isolado por tenant)
   - `PUT /api/v1/shelves/{id}` → 200 OK

5. **Location CRUD** — `LocationController`:
   - `POST /api/v1/shelves/{shelfId}/locations` → 201 Created (código único por tenant globalmente)
   - `GET /api/v1/shelves/{shelfId}/locations` → 200 OK
   - `GET /api/v1/locations/{id}` → 200 OK ou 404 (isolado por tenant)
   - `GET /api/v1/locations?type={type}` → 200 OK (filtra por `LocationType` no tenant)
   - `PUT /api/v1/locations/{id}` → 200 OK
   - `PATCH /api/v1/locations/{id}/deactivate` → 204 No Content

6. **`fullAddress` auto-computado** — Ao criar uma Location, o service DEVE derivar o `fullAddress` como `{warehouseCode}/{zoneCode}/{aisleCode}/{shelfCode}/{locationCode}` (ex: `WH-001/ZN-01/A01/S02/L003`). Campo `NOT NULL` no banco; nunca aceito via request body.

7. **Validações de negócio** — Todas retornam 422 (BusinessException) ou 404 (ResourceNotFoundException):
   - Código duplicado dentro do escopo → 422 com mensagem clara
   - Entidade pai não encontrada ou pertencente a outro tenant → 404
   - Tentativa de criar filho de entidade inativa (warehouse/zone desativado) → 422
   - `LocationType` inválido → 400 (validado por `@NotNull` + enum)

8. **Segurança** — Todos os endpoints exigem JWT Keycloak válido com claim `tenant_id`:
   - Operações de leitura: `@PreAuthorize("hasAnyRole('WMS_ADMIN', 'WMS_OPERATOR')")`
   - Operações de escrita (POST/PUT/PATCH): `@PreAuthorize("hasRole('WMS_ADMIN')")`
   - Requisições sem JWT → 401 Unauthorized (gerenciado pelo Spring Security)
   - Requisições com role insuficiente → 403 Forbidden

9. **Suite de testes ≥ 40 novos testes** — Distribuídos entre:
   - Service unit tests (Mockito, sem Spring context): 4-6 testes por service = ~25 testes
   - Controller integration tests (extends `AbstractIntegrationTest`): 3-5 testes por controller = ~20 testes
   - Os 54 testes existentes (Stories 1.1-1.4) continuam passando após adição das novas classes

    @Transactional
    @RequiredArgsConstructor
    public class WarehouseService {

        private final WarehouseRepository warehouseRepository;

        public WarehouseResponse create(CreateWarehouseRequest request) {
            UUID tenantId = TenantContextHolder.getTenantId();
            // Valida unicidade: código único por tenant
            if (warehouseRepository.existsByTenantIdAndCode(tenantId, request.code())) {
                throw new BusinessException("Warehouse code already exists: " + request.code());
            }
            String actor = SecurityContextHolder.getContext().getAuthentication().getName();
            Warehouse warehouse = Warehouse.builder()
                .tenantId(tenantId)
                .code(request.code())
                .name(request.name())
                .address(request.address())
                .active(true)
                .createdBy(actor)
                .build();
            return toResponse(warehouseRepository.save(warehouse));
        }

        @Transactional(readOnly = true)
        public List<WarehouseResponse> findAll() {
            UUID tenantId = TenantContextHolder.getTenantId();
            return warehouseRepository.findByTenantId(tenantId)
                .stream().map(this::toResponse).toList();
        }

        @Transactional(readOnly = true)
        public WarehouseResponse findById(UUID id) {
            return toResponse(getByTenant(id));
        }

        public WarehouseResponse update(UUID id, UpdateWarehouseRequest request) {
            Warehouse warehouse = getByTenant(id);
            warehouse.setName(request.name());
            warehouse.setAddress(request.address());
            return toResponse(warehouseRepository.save(warehouse));
        }

        public void deactivate(UUID id) {
            Warehouse warehouse = getByTenant(id);
            warehouse.setActive(false);
            warehouseRepository.save(warehouse);

            UUID tenantId = TenantContextHolder.getTenantId();
            return warehouseRepository.findById(id)
                .filter(w -> w.getTenantId().equals(tenantId))
                .orElseThrow(() -> new ResourceNotFoundException("Warehouse not found: " + id));
        }

        private WarehouseResponse toResponse(Warehouse w) { /* ... mapeamento */ }

    ```java
    @Service
    @Transactional
    @RequiredArgsConstructor
    public class LocationService {

        private final LocationRepository locationRepository;
        private final ShelfRepository shelfRepository;
        private final AisleRepository aisleRepository;
        private final ZoneRepository zoneRepository;
        private final WarehouseRepository warehouseRepository;

        public LocationResponse create(UUID shelfId, CreateLocationRequest request) {
            UUID tenantId = TenantContextHolder.getTenantId();

            // 1. Valida que shelf pertence ao tenant
            Shelf shelf = shelfRepository.findById(shelfId)
                .filter(s -> s.getTenantId().equals(tenantId))
                .orElseThrow(() -> new ResourceNotFoundException("Shelf not found: " + shelfId));

            // 2. Valida unicidade do código de localização (global por tenant)
            if (locationRepository.existsByTenantIdAndCode(tenantId, request.code())) {
                throw new BusinessException("Location code already exists: " + request.code());
            }

            // 3. Computa fullAddress carregando a hierarquia
            String fullAddress = computeFullAddress(shelf, request.code(), tenantId);

            Location location = Location.builder()
                .tenantId(tenantId)
                .shelf(shelf)
                .code(request.code())
                .fullAddress(fullAddress)
                .type(request.type() != null ? request.type() : LocationType.STORAGE)
                .capacityUnits(request.capacityUnits())
                .capacityWeightKg(request.capacityWeightKg())
                .active(true)
                .build();

            return toResponse(locationRepository.save(location));
        }

        private String computeFullAddress(Shelf shelf, String locationCode, UUID tenantId) {
            // Carrega a cadeia hierárquica — todos os @Transactional garantem sessão ativa
            Aisle aisle = aisleRepository.findById(shelf.getAisle().getId())
                .filter(a -> a.getTenantId().equals(tenantId))
                .orElseThrow(() -> new ResourceNotFoundException("Aisle not found for shelf"));

            Zone zone = zoneRepository.findById(aisle.getZone().getId())
                .filter(z -> z.getTenantId().equals(tenantId))
                .orElseThrow(() -> new ResourceNotFoundException("Zone not found for aisle"));

            Warehouse warehouse = warehouseRepository.findById(zone.getWarehouse().getId())
                .filter(w -> w.getTenantId().equals(tenantId))
                .orElseThrow(() -> new ResourceNotFoundException("Warehouse not found for zone"));

            return String.format("%s/%s/%s/%s/%s",
                warehouse.getCode(), zone.getCode(),
                aisle.getCode(), shelf.getCode(), locationCode);
        }

        // ... findAllByShelf, findById, findByType, update, deactivate
    }
    ```

private String computeFullAddress(Shelf shelf, String locationCode, UUID tenantId) {
    // A cadeia deve ser carregada dentro da mesma transação (@Transactional)
    // Lazy loading é seguro dentro do contexto transacional
    Aisle aisle  = /* load via aisleRepository.findById(shelf.getAisle().getId()) */;
    Zone zone    = /* load via zoneRepository.findById(aisle.getZone().getId()) */;
    Warehouse wh = /* load via warehouseRepository.findById(zone.getWarehouse().getId()) */;

    return String.format("%s/%s/%s/%s/%s",
        wh.getCode(), zone.getCode(),
        aisle.getCode(), shelf.getCode(), locationCode);
}
