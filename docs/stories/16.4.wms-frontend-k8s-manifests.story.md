# Story 16.4: Kubernetes Manifests (wms-web)

## Status

Done

## Story

**As a** engenheiro DevOps,
**I want** manifestos Kubernetes completos para o `wms-web`,
**so that** possa fazer deploy no cluster local (`kind`) e em produção com `kubectl apply -k infra/k8s/wms-web/`.

## Complexity

**Estimativa:** M (Medium) · ~1 dia

## Dependencies

- Story 16.3 ✅ (imagem disponível em ghcr.io)

## Out of Scope

- ❌ TLS/cert-manager (produção — futuro)
- ❌ HPA (Horizontal Pod Autoscaler — futuro)
- ❌ Health endpoints em `/api/health` (Story 16.5 — probes usam `/` temporariamente)

## Acceptance Criteria

1. **`infra/k8s/wms-web/`** com 7 arquivos Kustomize:
   - `kustomization.yaml`, `namespace.yaml`, `configmap.yaml`
   - `secret.yaml` (template — sem valores reais), `deployment.yaml`
   - `service.yaml` (ClusterIP), `ingress.yaml` (nginx)
2. **Deployment:** 2 réplicas (1 para kind local), rolling update zero-downtime, resource limits definidos
3. **ConfigMap:** variáveis de ambiente não-sensíveis (WMS_API_URL, NEXT_PUBLIC_*)
4. **Secret template:** instrução clara — criar com `kubectl create secret`, NÃO commitar valores
5. **Ingress:** `wms.local` → Service port 80 (nginx ingress controller)
6. **`kubectl apply -k infra/k8s/wms-web/`** sem erros (dry-run)
7. **`infra/k8s/README.md`** com quick-start para kind local

### kind Quick Start

```bash
kind create cluster --name odin
kubectl apply -f https://raw.githubusercontent.com/kubernetes/ingress-nginx/main/deploy/static/provider/kind/deploy.yaml
kubectl apply -k infra/k8s/wms-web/
# Adicionar ao /etc/hosts: 127.0.0.1 wms.local
```
