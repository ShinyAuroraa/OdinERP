## Story

**As a** desenvolvedor,
**I want** um projeto Next.js 15 estruturado com design system configurado,
**so that** o desenvolvimento do frontend parta de uma base consistente com os padrões visuais e técnicos do ODIN ERP.

## Acceptance Criteria

1. Next.js 15 (App Router) + React 19 + TypeScript com estrutura de pastas definida: `app/`, `components/`, `lib/` (contendo `hooks/`, `api/`, `stores/`, `validations/`), `types/`
2. Tailwind CSS + shadcn/ui instalados e configurados com tema base
3. Layout principal criado: sidebar + header + área de conteúdo (`components/layout/`)
4. Páginas de loading e 404 customizadas (`app/loading.tsx`, `app/not-found.tsx`)
5. ESLint (`@typescript-eslint/recommended-type-checked`) + Prettier configurados e `pnpm lint` sem erros
6. Dockerfile multi-stage para produção (Next.js `output: 'standalone'`, `node:20-alpine`, user não-root)
7. Variáveis de ambiente tipadas via `lib/env.ts` com validação Zod + `.env.example` documentando todas as vars necessárias

### Dockerfile
[Source: architecture.md#Section-13.4]

Multi-stage build com `node:20-alpine`:
1. **deps:** instala dependências pnpm (cache layer)
2. **builder:** executa `pnpm build` (inclui Next.js standalone output)
3. **runner:** copia apenas `.next/standalone` + assets — user não-root (`nextjs:1001`)

`next.config.ts` **DEVE** ter `output: 'standalone'` para o Docker funcionar.

### Coding Standards TypeScript/React
[Source: architecture.md#Section-16.2]

- `PascalCase` para componentes, `useX` hooks, `xStore` stores, `xSchema` Zod schemas
- **Server Components** por default — `'use client'` apenas quando necessário (estado, event handlers)
- Props interface co-localizada com o componente
- **Sem `any` type** — ESLint enforça

### Debug Log References

- **FIX-01:** `pnpm create next-app@latest` instalou Next.js 16.1.6 + Tailwind 4. Revertido para `next@15` via `pnpm create next-app@15` → Next.js 15.5.12 instalado. Tailwind 4 ainda presente no template Next.js 15 (comportamento esperado em 2026).
- **FIX-02:** Tailwind 4 não usa `tailwind.config.ts` — usa CSS-based config (`@import "tailwindcss"` + `@theme inline {}`). shadcn/ui auto-detectou Tailwind 4 e configurou corretamente. Task 3.3 adaptada.
- **FIX-03:** ESLint `recommendedTypeChecked` falhava em `.mjs` config files (não estão em `tsconfig.json`). Fix: aplicar `recommendedTypeChecked` apenas a `files: ['**/*.ts', '**/*.tsx']`.
- **FIX-04:** `pnpm build` falhava com `EPERM: symlink` — Windows + pnpm symlinks + Next.js standalone. Fix: `.npmrc` com `node-linker=hoisted`, reinstall, build OK.
- **FIX-05:** Route conflict — `app/page.tsx` e `app/(dashboard)/page.tsx` ambos resolvem para `/`. Fix: página movida para `app/(dashboard)/dashboard/page.tsx` → URL `/dashboard`.

### Completion Notes List

- **Tailwind 4:** Next.js 15 template (2026) usa Tailwind CSS 4.2.0 em vez de 3.4+. shadcn/ui tem suporte nativo a Tailwind 4. Config via CSS vars em `globals.css`, não via `tailwind.config.ts`.
- **Zod 4:** Zod 4.3.6 instalado (spec dizia "zod" sem versão). API compatível com uso da story (`z.string().url()`, `z.string().min()`, `z.object().parse()`).
- **pnpm node-linker=hoisted:** Necessário para `pnpm build` no Windows com `output: 'standalone'`. Adicionado `.npmrc` ao projeto.
