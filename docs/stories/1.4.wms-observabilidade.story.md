# Story 1.4: Observabilidade WMS (Prometheus + ELK + Jaeger + Istio)

## Status

Done

## Story

**As a** engenheiro de plataforma,
**I want** configurar métricas customizadas WMS no Prometheus, logs JSON estruturados para o ELK Stack, distributed tracing com Jaeger via OpenTelemetry e anotações Istio nos manifests Kubernetes,
**so that** toda operação do WMS seja observável em produção: métricas de negócio visíveis no Grafana, logs correlacionados com trace ID no Kibana, requisições rastreadas de ponta a ponta no Jaeger e o service mesh Istio capaz de injetar sidecar automaticamente.

## Business Value

**Por que esta story é crítica:**

- **Production-readiness:** Sem observabilidade, não há visibilidade de problemas em produção. Esta story é requisito do SLA de 99,9% de uptime.
- **Debugging multi-tenant:** Com `tenant_id` no MDC de cada log e trace, é possível isolar logs de um tenant específico no Kibana em segundos.
- **Capacity planning:** Métricas de `wms.stock.movements.total` e `wms.api.latency` alimentam o autoscaling (HPA baseado em custom metrics futuramente).
- **Correlation entre logs e traces:** `traceId` + `spanId` no log JSON permite navegar do log de erro diretamente para o trace do Jaeger — reduz MTTR drasticamente.

## Risks

| Risco | Probabilidade | Impacto | Mitigação |
|-------|--------------|---------|-----------|
| `micrometer-tracing-bridge-otel` conflito de versão com Spring Boot 3.2.3 | Baixa | Alto | Usar versão gerenciada pelo BOM; não especificar versão manualmente |
| Logstash encoder quebrar testes de log existentes | Média | Baixo | Profile `test` pode usar pattern simples; logback-spring.xml com `springProfile` |
| Istio sidecar annotation causar falha no startup local (sem Istio) | Baixa | Baixo | Annotation é apenas metadata K8s — não afeta execução local fora do cluster |
| Sampling rate 100% em prod sobrecarregar Jaeger | Média | Médio | Default 10% em prod via `${TRACING_SAMPLING_PROBABILITY:0.1}`; 100% apenas em dev/test |
| `TenantContextHolder.getTenantId()` null no MDC para requests sem auth | Baixa | Baixo | Verificar null antes de setar MDC; `tenantId: N/A` como fallback |

## Acceptance Criteria

1. **Métricas customizadas WMS registradas** — Bean `WmsMetrics` criado em `com.odin.wms.config.metrics`. Métricas registradas no `MeterRegistry`:
   - `wms.stock.movements.total` (Counter) — total de movimentações de estoque processadas
   - `wms.operations.errors.total` (Counter) — total de erros em operações WMS, com tag `operation`
   - `wms.api.request.duration` (Timer) — latência das requisições, com tags `method`, `endpoint`, `status`
   - `wms.active.tenants` (Gauge) — número de tenants com requisições ativas (via `AtomicInteger`)
   - Endpoint `/actuator/prometheus` retorna as novas métricas com prefixo `wms_`

2. **Logs JSON estruturados** — `logback-spring.xml` configurado com `LogstashEncoder`:
   - Formato JSON em todos os ambientes não-test
   - Campos obrigatórios em cada linha de log: `timestamp`, `level`, `logger`, `message`, `traceId`, `spanId`, `tenantId`, `application`, `environment`
   - `tenantId` populado via MDC a partir do `TenantContextHolder` no `TenantMdcFilter`
   - Profile `test` usa encoder de texto simples (legibilidade nos testes)

3. **TenantMdcFilter criado** — Novo filtro `TenantMdcFilter` (após `TenantContextFilter` na cadeia):
   - Lê `TenantContextHolder.getTenantId()` e faz `MDC.put("tenantId", tenantId.toString())`
   - `finally { MDC.remove("tenantId"); }` para evitar contaminação entre threads
   - Registrado em `SecurityConfig` após `TenantContextFilter`

4. **Distributed tracing configurado (Micrometer + OpenTelemetry)** — Dependências adicionadas ao `pom.xml`:
   - `micrometer-tracing-bridge-otel` (versão gerenciada pelo BOM)
   - `opentelemetry-exporter-otlp` (versão gerenciada pelo BOM)
   - `application.yml` configurado:
     - `management.tracing.sampling.probability: ${TRACING_SAMPLING_PROBABILITY:1.0}`
     - `management.otlp.tracing.endpoint: ${JAEGER_OTLP_ENDPOINT:http://localhost:4318/v1/traces}`
   - `traceId` e `spanId` automaticamente propagados via MDC pelo Micrometer Tracing
   - W3C TraceContext headers (`traceparent`) propagados em requisições de saída

5. **Testes de observabilidade passando** — Suite de testes com ≥ 6 testes:
   - `WmsMetricsTest` (unit): verifica que todos os 4 tipos de métricas são registrados no `SimpleMeterRegistry`
   - `TenantMdcFilterTest` (unit): verifica que `tenantId` é inserido/removido corretamente do MDC
   - `ObservabilityIntegrationTest` (integration): verifica que `/actuator/prometheus` retorna HTTP 200 sem auth (management port) e contém `wms_` métricas
   - Testes existentes (43/43) continuam passando após adição das novas dependências

6. **K8s manifests atualizados** — Deployment `k8s/wms/deployment.yaml` com:
   - Annotation `sidecar.istio.io/inject: "true"` no `spec.template.metadata.annotations`
   - Annotation `prometheus.io/scrape: "true"`, `prometheus.io/port: "8081"`, `prometheus.io/path: "/actuator/prometheus"` para scraping automático

    ```java
    /**
     * Popula o MDC com o tenantId do TenantContextHolder para correlação de logs.
     * Deve rodar APÓS TenantContextFilter (que já populou o TenantContextHolder).
     * NOT @Component — registrado manualmente em SecurityConfig.
     */
    public class TenantMdcFilter extends OncePerRequestFilter {

        private static final String MDC_TENANT_ID_KEY = "tenantId";

        @Override
        protected void doFilterInternal(HttpServletRequest request,
                                        HttpServletResponse response,
                                        FilterChain filterChain) throws ServletException, IOException {
            try {
                UUID tenantId = TenantContextHolder.getTenantId();
                if (tenantId != null) {
                    MDC.put(MDC_TENANT_ID_KEY, tenantId.toString());
                }
                filterChain.doFilter(request, response);
            } finally {
                MDC.remove(MDC_TENANT_ID_KEY);
            }
        }
    }
    ```

    ```java
    /**
     * Métricas customizadas do WMS registradas no MeterRegistry.
     * Nomenclatura: wms.<domínio>.<operação>[.<unidade>]
     * Convenção Prometheus: snake_case quando exportado (wms_stock_movements_total)
     */
    @Component
    public class WmsMetrics {

        public static final String STOCK_MOVEMENTS = "wms.stock.movements";
        public static final String OPERATION_ERRORS = "wms.operations.errors";
        public static final String API_REQUEST_DURATION = "wms.api.request.duration";
        public static final String ACTIVE_TENANTS_GAUGE = "wms.active.tenants";

        private final Counter stockMovementsCounter;
        private final AtomicInteger activeTenantsCount;

        public WmsMetrics(MeterRegistry registry) {
            this.stockMovementsCounter = Counter.builder(STOCK_MOVEMENTS)
                    .description("Total de movimentações de estoque processadas")
                    .tag("service", "wms")
                    .register(registry);

            this.activeTenantsCount = registry.gauge(
                    ACTIVE_TENANTS_GAUGE,
                    Tags.of("service", "wms"),
                    new AtomicInteger(0));

            // Timer e Counter de erros são criados por operação — registrados pelos services futuros
            // Registramos aqui com tags base para garantir que apareçam no /actuator/prometheus
            Counter.builder(OPERATION_ERRORS)
                    .description("Total de erros em operações WMS")
                    .tag("operation", "unknown")
                    .register(registry);

            Timer.builder(API_REQUEST_DURATION)
                    .description("Duração das requisições às APIs WMS")
                    .tag("endpoint", "unknown")
                    .tag("method", "unknown")
                    .tag("status", "unknown")
                    .register(registry);
        }

        public void incrementStockMovements() {
            stockMovementsCounter.increment();
        }

        public void incrementStockMovements(double amount) {
            stockMovementsCounter.increment(amount);
        }

        public AtomicInteger getActiveTenantsGauge() {
            return activeTenantsCount;
        }
    }
    ```

### Padrão TenantMdcFilter

```java
package com.odin.wms.config.security;

// NOT @Component — mesmo padrão do TenantContextFilter
// Evita double-registration como Servlet filter + Spring Security filter
public class TenantMdcFilter extends OncePerRequestFilter {

    @Override
    protected void doFilterInternal(HttpServletRequest request,
                                    HttpServletResponse response,
                                    FilterChain filterChain) throws ServletException, IOException {
        try {
            UUID tenantId = TenantContextHolder.getTenantId();
            if (tenantId != null) {
                MDC.put("tenantId", tenantId.toString());
            }
            filterChain.doFilter(request, response);
        } finally {
            MDC.remove("tenantId");  // Crítico: evitar vazamento entre requisições
        }
    }
}
```
