# Story 4.3: Inventário Físico (Contagem de Estoque)

## Status

Done

## Story

**As a** supervisor de armazém (WMS Supervisor / WMS Admin),
**I want** criar e gerenciar sessões de inventário físico — cíclico (por zona/corredor) ou geral — com geração automática de listas de contagem, registro de quantidades contadas por operadores, reconciliação automática de divergências, opção de dupla contagem, e aprovação formal de ajustes acima do threshold configurável,
**so that** o saldo de estoque no sistema seja consistente com a realidade física do armazém, com rastreabilidade completa de cada ajuste realizado e publicação de eventos para MRP/SCM.

## Risks

| Risco | Probabilidade | Impacto | Mitigação |
|-------|--------------|---------|-----------|
| Race condition entre contagem e movimentações ativas | Média | Alto | `InventoryCount.status = IN_PROGRESS` bloqueia logicamente ajustes — aviso, não lock duro |
| StockItem.quantity diverge após ajuste (optimistic locking) | Média | Alto | `@Version` em `StockItem` — retry em `ObjectOptimisticLockingFailureException` → HTTP 409 |
| Lista de contagem muito grande para armazéns grandes | Média | Médio | Paginação na query de `InventoryCountItem`; índice em `(inventory_count_id, location_id)` |
| Dupla contagem inconsistente com primeira contagem | Baixa | Médio | Só habilitar segunda contagem em itens com divergência; `secondCountedQty` obrigatório se status = SECOND_COUNT |
| Redis cache stale após ajuste de inventário | Alta | Médio | Reutilizar `@CacheEvict` de StockBalanceService — já testado em Story 4.1 |

## Acceptance Criteria

### AC1 — POST /inventory/count (criar sessão de inventário)
**Given** um usuário autenticado com role `WMS_SUPERVISOR` ou `WMS_ADMIN`,
**When** chama `POST /inventory/count` com body:
```json
{
  "countType": "CYCLIC",
  "warehouseId": "uuid",
  "zoneId": "uuid",
  "adjustmentThreshold": 5
}
```
**Then** retorna `201 Created` com:
```json
{
  "countId": "uuid",
  "countType": "CYCLIC",
  "warehouseId": "uuid",
  "zoneId": "uuid",
  "status": "DRAFT",
  "adjustmentThreshold": 5,
  "totalItems": 48,
  "createdAt": "2026-02-22T10:00:00Z"
}
```
- `countType` aceita `CYCLIC` (escopo por `zoneId` obrigatório) ou `FULL` (escopo = armazém inteiro, `zoneId` null)
- `adjustmentThreshold` = percentual máximo de divergência sem aprovação (ex.: 5 = 5%); default 0 (toda divergência requer aprovação)
- `totalItems` = número de `InventoryCountItem` gerados automaticamente (uma entrada por StockItem ativo no escopo)
- Status inicial: `DRAFT`

### AC2 — POST /inventory/count/{countId}/start (iniciar contagem)
**Given** `InventoryCount` em status `DRAFT`,
**When** chama `POST /inventory/count/{countId}/start`,
**Then**:
- Status muda para `IN_PROGRESS`
- `startedAt` preenchido com timestamp atual
- Retorna `200 OK` com `InventoryCountResponse` atualizado

### AC3 — GET /inventory/count/{countId}/items (lista de contagem)
**Given** `InventoryCount` existente no tenant,
**When** chama `GET /inventory/count/{countId}/items?page=0&size=50`,
**Then** retorna `200 OK` com lista paginada de `InventoryCountItemResponse`:
```json
{
  "content": [
    {
      "itemId": "uuid",
      "locationCode": "A-01-01",
      "productCode": "SKU-001",
      "lotNumber": "LOT-001",
      "expiryDate": "2026-12-31",
      "expectedQty": 100,
      "countedQty": null,
      "status": "PENDING"
    }
  ],
  "totalElements": 48,
  "page": 0,
  "size": 50
}
```
- `expectedQty` = `StockItem.quantity` no momento da criação do inventário
- `countedQty` null até o operador submeter

### AC4 — PATCH /inventory/count/{countId}/items/{itemId} (submeter contagem)
**Given** um usuário autenticado com role `WMS_OPERATOR` ou superior e `InventoryCount` em status `IN_PROGRESS`,
**When** chama `PATCH /inventory/count/{countId}/items/{itemId}` com:
```json
{ "countedQty": 95 }
```
**Then**:
- `InventoryCountItem.countedQty = 95`, status = `COUNTED`
- `divergenceQty = countedQty - expectedQty = -5`
- `divergencePct = abs(divergenceQty / expectedQty) * 100 = 5.0%`
- Retorna `200 OK` com item atualizado incluindo `divergenceQty` e `divergencePct`
- Operador sem role WMS → `403 Forbidden` ← **CONCERN-1 de Story 4.2 endereçado aqui**

### AC5 — POST /inventory/count/{countId}/reconcile (reconciliação)
**Given** todos os itens com status `COUNTED` (ou segunda contagem concluída),
**When** chama `POST /inventory/count/{countId}/reconcile`,
**Then**:
- Status muda para `RECONCILED`
- Itens com `abs(divergencePct) <= adjustmentThreshold` → status `AUTO_APPROVED` (não requerem aprovação manual)
- Itens com `abs(divergencePct) > adjustmentThreshold` → status `PENDING_APPROVAL`
- Retorna `200 OK` com resumo:
```json
{
  "totalItems": 48,
  "autoApproved": 40,
  "pendingApproval": 8,
  "noVariance": 35,
  "status": "RECONCILED"
}
```

### AC6 — Segunda Contagem (double count para divergências)
**Given** `InventoryCount` em status `RECONCILED` com itens `PENDING_APPROVAL`,
**When** chama `POST /inventory/count/{countId}/items/{itemId}/second-count` com:
```json
{ "secondCountedQty": 96 }
```
**Then**:
- `InventoryCountItem.secondCountedQty = 96`
- `adjustedQty` = média das duas contagens (ou a segunda, se configurado)
- Status do item = `SECOND_COUNTED`
- Divergência recalculada com base em `adjustedQty`
- Supervisor decide se aprova após segunda contagem

### AC7 — POST /inventory/count/{countId}/approve (aprovação de ajustes)
**Given** `InventoryCount` em status `RECONCILED` ou `PENDING_APPROVAL` e usuário com role `WMS_SUPERVISOR` ou `WMS_ADMIN`,
**When** chama `POST /inventory/count/{countId}/approve`,
**Then**:
- Para cada item `PENDING_APPROVAL`:
  - `StockItem.quantity` atualizado com `adjustedQty` (ou `countedQty` se sem segunda contagem)
  - `StockMovement` criado com `type = INVENTORY_ADJUSTMENT`, `quantity = adjustedQty - expectedQty`
  - `TraceabilityIndexer.indexMovementAsync()` chamado para cada ajuste
  - Cache Redis evicted: `@CacheEvict(value = "stockBalance", key = "...")` para cada `locationId` afetado
- Para itens `AUTO_APPROVED` (divergência dentro do threshold): mesmo fluxo de ajuste
- `InventoryCount.status = APPROVED`, `approvedBy = userId atual`
- Retorna `200 OK` com resumo dos ajustes aplicados

### AC8 — POST /inventory/count/{countId}/close (fechar inventário)
**Given** `InventoryCount` em status `APPROVED`,
**When** chama `POST /inventory/count/{countId}/close`,
**Then**:
- Status = `CLOSED`, `closedAt` preenchido
- Retorna `200 OK`
- Inventário fechado não pode ser reaberto

### AC9 — RBAC e isolamento multi-tenant
**Given** endpoints de inventário,
**Then**:
- `WMS_OPERATOR`: pode `GET` lista e `PATCH` itens (submeter contagem)
- `WMS_SUPERVISOR` e `WMS_ADMIN`: podem criar, iniciar, reconciliar, aprovar e fechar
- Token válido **sem nenhuma role WMS** (ex.: `ROLE_FINANCE`) → `403 Forbidden`
- Sem token → `401 Unauthorized`
- Tenant isolation: `InventoryCount` sempre filtrado por `tenantId` do JWT
- `InventoryCount` de outro tenant → `404 Not Found`

CREATE TABLE inventory_counts (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    tenant_id UUID NOT NULL,
    warehouse_id UUID NOT NULL REFERENCES warehouses(id),
    zone_id UUID REFERENCES zones(id),
    count_type VARCHAR(20) NOT NULL CHECK (count_type IN ('CYCLIC', 'FULL')),
    status VARCHAR(30) NOT NULL DEFAULT 'DRAFT',
    adjustment_threshold INTEGER NOT NULL DEFAULT 0,
    started_at TIMESTAMP WITH TIME ZONE,
    closed_at TIMESTAMP WITH TIME ZONE,
    approved_by UUID,
    created_at TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW(),
    created_by UUID,
    CONSTRAINT chk_cyclic_zone CHECK (count_type != 'CYCLIC' OR zone_id IS NOT NULL)
);

CREATE TABLE inventory_count_items (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    inventory_count_id UUID NOT NULL REFERENCES inventory_counts(id),
    tenant_id UUID NOT NULL,
    location_id UUID NOT NULL REFERENCES locations(id),
    product_id UUID NOT NULL REFERENCES products_wms(id),
    lot_id UUID REFERENCES lots(id),
    expected_qty NUMERIC(15,3) NOT NULL,
    counted_qty NUMERIC(15,3),
    second_counted_qty NUMERIC(15,3),
    adjusted_qty NUMERIC(15,3),
    divergence_qty NUMERIC(15,3),
    divergence_pct NUMERIC(8,4),
    status VARCHAR(30) NOT NULL DEFAULT 'PENDING',
    created_at TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW(),
    CONSTRAINT uq_count_location_product_lot
        UNIQUE (inventory_count_id, location_id, product_id, lot_id)
);

CREATE INDEX idx_inv_counts_tenant_status ON inventory_counts(tenant_id, status);
CREATE INDEX idx_inv_items_count_status ON inventory_count_items(inventory_count_id, status);
CREATE INDEX idx_inv_items_count_location ON inventory_count_items(inventory_count_id, location_id);
