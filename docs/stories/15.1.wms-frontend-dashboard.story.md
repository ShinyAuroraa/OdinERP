# Story 15.1: Dashboard & Analytics — Visão Geral Operacional WMS

## Status

Done

## Story

**As a** supervisor de armazém (WMS_SUPERVISOR),
**I want** visualizar um dashboard com métricas operacionais chave do WMS,
**so that** possa monitorar a saúde do armazém, identificar gargalos e tomar decisões rápidas.

## Complexity

**Estimativa:** M (Medium) · ~2-3 dias

## Dependencies

- Story 9.1 ✅, 9.2 ✅ (fundação)
- Todas as stories anteriores ✅ (dados disponíveis via APIs existentes)

## Out of Scope

- ❌ Gráficos interativos animados (recharts/d3)
- ❌ Relatórios exportáveis do dashboard
- ❌ Dashboard customizável pelo usuário

## Acceptance Criteria

1. **Página `/dashboard`** — visão geral operacional:
   - 4 KPI cards: Total Armazéns, Total Produtos, Picking Pendente, Alertas MRP
   - Seção "Atividade Recente": últimas 5 entradas de auditoria
   - Seção "Status dos Processos": contagem por status (Recebimento, Picking, Packing, Shipping)
   - staleTime 30s (dashboard atualiza frequentemente)

2. **React Query hooks** — `lib/api/dashboard.ts`:
   - `useDashboardStats()` — agrega dados de múltiplos endpoints existentes
   - Sem novos endpoints backend — usa APIs existentes

3. **TypeScript types** — `types/dashboard.ts`:
   - `DashboardStats`, `KpiCard`, `ProcessStatus`

4. **Sidebar**: rota `/dashboard` → `implemented: true`

5. **Testes unitários** (≥ 6 novos):
   - Renderiza KPI cards
   - Exibe estado de carregamento
   - Exibe dados de auditoria recente
   - Exibe alertas MRP
   - Renderiza status dos processos
