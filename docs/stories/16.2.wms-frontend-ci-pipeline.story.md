# Story 16.2: GitHub Actions CI Pipeline

## Status

Done

## Story

**As a** engenheiro DevOps,
**I want** um pipeline de CI automático no GitHub Actions para o `wms-web`,
**so that** todo PR e push para `main` seja validado automaticamente (lint + typecheck + test + build) antes de ser mergeado.

## Complexity

**Estimativa:** M (Medium) · ~0.5 dia

## Dependencies

- Story 16.1 ✅ (Dockerfile — `output: 'standalone'` ativo)
- GitHub Actions disponível no repositório

## Out of Scope

- ❌ CD (deploy) — Story 16.3
- ❌ Docker build no CI — Story 16.3
- ❌ Branch protection rules (manual no GitHub)

## Acceptance Criteria

1. **`.github/workflows/ci.yml`** disparado em push + PR para `main`:
   - paths filter: `apps/wms-web/**` + `.github/workflows/ci.yml`
2. **Jobs sequenciais**: install → lint → typecheck → test → build
3. **Node.js 20** com cache npm (cache-dependency-path para `apps/wms-web/`)
4. **Variáveis de ambiente** de build (NEXT_TELEMETRY_DISABLED, NEXT_PUBLIC_*)
5. **working-directory**: `apps/wms-web` em todos os steps
6. **Falha em qualquer job** → bloqueia merge do PR
