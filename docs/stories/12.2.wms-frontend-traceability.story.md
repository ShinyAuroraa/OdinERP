# Story 12.2: Rastreabilidade — Histórico de Lotes + Árvore de Rastreabilidade + GS1

## Status

Done

## Story

**As a** supervisor de armazém (WMS_SUPERVISOR),
**I want** rastrear o histórico completo de movimentos de lotes e números de série, visualizar a árvore de rastreabilidade e fazer parse de códigos GS1,
**so that** possa realizar auditorias de qualidade e responder a recalls de produtos rapidamente.

## Complexity

**Estimativa:** M (Medium) · ~3-4 dias

## Dependencies

- Story 9.1 ✅, 9.2 ✅
- Backend 4.2 Done ✅ — `/api/v1/traceability`, `/api/v1/gs1`

## Out of Scope

- ❌ Scanner de código de barras via câmera
- ❌ Integração com sistema de recalls externo

## Acceptance Criteria

1. **Página `/traceability`** — busca de rastreabilidade:
   - Form de busca: tipo (Lote / Número de Série), valor
   - Resultado: timeline de movimentos (data, tipo, origem→destino, quantidade, operador)
   - Botão "Ver Árvore" → expande TraceabilityTree component

2. **Visualização de lotes por validade** (aba FEFO):
   - DataTable: Produto, Lote, Data Validade, Quantidade, Status (válido/vencendo/vencido)
   - Filtros: warehouseId, expiryBefore (datepicker)
   - Badge de alerta para lotes vencendo em ≤ 30 dias

3. **Parser GS1** (aba GS1):
   - Input para código GS1 (texto)
   - Resultado: GTIN, lote, serial, datas (fabricação/validade), SSCC
   - Botão "Gerar Código": campos GTIN + lotNumber + serial → gera GS1

4. **React Query hooks** — `lib/api/traceability.ts`:
   - `useLotTraceability(lotNumber)` — GET /traceability/lot/{lotNumber}
   - `useSerialTraceability(serialNumber)` — GET /traceability/serial/{serialNumber}
   - `useTraceabilityTree(lotId)` — GET /traceability/lot/{lotId}/tree
   - `useExpiryAlerts(filters)` — GET /traceability/product/{productId}/expiry
   - `useParseGS1` — POST /gs1/parse (mutation)
   - `useGenerateGS1` — GET /gs1/generate

5. **TypeScript types** — `types/traceability.ts`

6. **Testes unitários** (≥ 6 novos)
