# Story 12.3: Inventário Físico — Sessões de Contagem + Reconciliação + Aprovação

## Status

Done

## Story

**As a** supervisor de armazém (WMS_SUPERVISOR),
**I want** criar e gerenciar sessões de inventário físico com fluxo completo (criar → iniciar → contar → reconciliar → aprovar → fechar),
**so that** o estoque real do armazém seja validado periodicamente e ajustes sejam aprovados de forma controlada.

## Complexity

**Estimativa:** L (Large) · ~4-5 dias

## Dependencies

- Story 9.1 ✅, 9.2 ✅
- Backend 4.3 Done ✅ — `/api/v1/inventory`

## Out of Scope

- ❌ Scanner de código de barras para contagem
- ❌ Geração de relatório PDF

## Acceptance Criteria

1. **Página `/inventory`** — lista de sessões de inventário:
   - DataTable: ID, Armazém, Data Criação, Status, Itens Totais, Divergências, Ações
   - Botão "Nova Sessão" (WMS_SUPERVISOR)
   - Status: CREATED/IN_PROGRESS/RECONCILED/APPROVED/CLOSED

2. **Detalhe da sessão** `/inventory/{countId}`:
   - Status machine visual: Criada → Em Contagem → Reconciliada → Aprovada → Fechada
   - Lista de itens paginada (produto, localização, lote, qtd esperada, qtd contada, divergência)
   - Para cada item PENDING: botão "Contar" → modal com input quantidade
   - Botão "Reconciliar" (após todos itens contados) → POST /reconcile
   - Card de resumo de reconciliação: total itens, divergências, valor ajuste
   - Botão "Aprovar Ajustes" (SUPERVISOR) → POST /approve
   - Botão "Fechar" → POST /close

3. **React Query hooks** — `lib/api/inventory.ts`:
   - `useInventoryCounts(filters?)` — GET /api/v1/inventory/count
   - `useInventoryCount(countId)` — GET individual
   - `useCreateInventoryCount` — POST /count
   - `useStartCount(countId)` — POST /start
   - `useCountItems(countId, page)` — GET /count/{id}/items
   - `useSubmitCountItem(countId, itemId)` — PATCH /items/{id}
   - `useReconcile(countId)` — POST /reconcile
   - `useApproveCount(countId)` — POST /approve
   - `useCloseCount(countId)` — POST /close

4. **TypeScript types** — `types/inventory.ts`

5. **Proteção por role**: SUPERVISOR para criar/aprovar/fechar; OPERATOR para contar

6. **Testes unitários** (≥ 8 novos)
