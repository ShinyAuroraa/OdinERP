# Story 11.2: Putaway & Quarentena — Alocação de Localizações + Gestão de Qualidade

## Status

Done

## Story

**As a** operador de armazém (WMS_OPERATOR),
**I want** gerenciar tarefas de putaway (alocação de mercadorias recebidas em localizações) e quarentena (revisão de qualidade com decisão de liberar/devolver/sucatar) via interface web,
**so that** o fluxo inbound seja concluído com as mercadorias alocadas nas localizações corretas e itens de qualidade duvidosa tratados adequadamente.

## Complexity

**Estimativa:** M (Medium) · ~3-4 dias de desenvolvimento
[Duas entidades independentes: Putaway tasks + Quarantine tasks com workflows próprios]

## Acceptance Criteria

1. **Página `/putaway`** — fila de tarefas de putaway:
   - DataTable: Produto, Lote, Quantidade, Localização Sugerida, Status, Ações
   - Filtro por status (PENDING/IN_PROGRESS/COMPLETED/CANCELLED) + armazém
   - Botão "Iniciar" por tarefa (WMS_OPERATOR) → `PATCH /putaway-tasks/{id}/start`
   - Modal de confirmação: confirmar localização sugerida ou escolher outra (LocationSelect)
   - Confirmar → `PATCH /putaway-tasks/{id}/confirm` com `confirmedLocationId`
   - Loading skeleton + empty state "Nenhuma tarefa de putaway pendente"

2. **Página `/quarantine`** — fila de tarefas de quarentena:
   - DataTable: Produto, Lote, Quantidade, Armazém, Status, Ações
   - Filtro por status (PENDING/IN_PROGRESS/DECIDED/CANCELLED)
   - Botão "Iniciar Revisão" (WMS_SUPERVISOR) → `PATCH /quarantine-tasks/{id}/start`
   - Modal de decisão (apenas SUPERVISOR):
     - Select: RELEASE_TO_STOCK / RETURN_TO_SUPPLIER / SCRAP
     - Textarea: observações de qualidade
     - Confirmar → `PATCH /quarantine-tasks/{id}/decide`
   - Loading skeleton + empty state "Nenhuma tarefa de quarentena pendente"

3. **Status badges** para ambas as filas:
   - PENDING → cinza, IN_PROGRESS → azul, COMPLETED/DECIDED → verde, CANCELLED → vermelho

4. **React Query hooks** — `lib/api/putaway.ts`:
   - `usePutawayTasks(filters?)` — GET /api/v1/putaway-tasks, staleTime 30s
   - `useStartPutaway(id)` — PATCH /start
   - `useConfirmPutaway(id)` — PATCH /confirm
   - `useCancelPutaway(id)` — PATCH /cancel (SUPERVISOR)
   - `useQuarantineTasks(filters?)` — GET /api/v1/quarantine-tasks, staleTime 30s
   - `useStartQuarantine(id)` — PATCH /start
   - `useDecideQuarantine(id)` — PATCH /decide
   - `useCancelQuarantine(id)` — PATCH /cancel

5. **TypeScript types** — `types/putaway.ts`:
   - `PutawayTask`, `PutawayTaskStatus`, `QuarantineTask`, `QuarantineTaskStatus`
   - `ConfirmPutawayRequest`, `DecideQuarantineRequest`, `QuarantineDecision`

6. **Proteção por role**:
   - Putaway (iniciar + confirmar): WMS_OPERATOR+
   - Quarentena (iniciar + decidir): WMS_SUPERVISOR+
   - Cancelar ambos: WMS_SUPERVISOR+
   - Botões restritos ocultados para roles sem permissão

7. **Testes unitários** (≥ 10 novos):
   - PutawayList: render, loading, empty, filtro status
   - QuarantineList: render, loading, empty, botão decidir oculto para OPERATOR
   - PutawayConfirmModal: render localização sugerida, submit
   - Hooks: usePutawayTasks mock
