# Story 12.4: Auditoria & Transferências Internas — Log Imutável + Movimentação entre Localizações

## Status

Done

## Story

**As a** supervisor de armazém (WMS_SUPERVISOR),
**I want** visualizar o log de auditoria imutável de todas as operações e gerenciar transferências internas entre localizações,
**so that** tenha rastreabilidade completa de ações e possa mover estoque entre localizações de forma controlada.

## Complexity

**Estimativa:** M (Medium) · ~3-4 dias

## Dependencies

- Story 9.1 ✅, 9.2 ✅
- Backend 4.4 Done ✅ — `/api/v1/audit`
- Backend 4.5 Done ✅ — `/api/v1/transfers`

## Out of Scope

- ❌ Export PDF do audit log
- ❌ Alertas automáticos por anomalias

## Acceptance Criteria

1. **Página `/audit`** — log de auditoria:
   - DataTable: Data/Hora, Entidade, Tipo de Ação, Usuário, Detalhes (expandível)
   - Filtros: entityType (select), from/to (datepicker range)
   - Botão "Exportar JSON" (ADMIN) → GET /audit/log/export
   - Paginação (backend paginado, page/size)

2. **Configuração de retenção** (aba Configurações, ADMIN):
   - Card com retenção atual (dias)
   - Form para atualizar PUT /audit/retention-config

3. **Página `/transfers`** — transferências internas:
   - DataTable: Data, Origem, Destino, Produto, Lote, Qtd, Status, Ações
   - Filtro por status (PENDING/CONFIRMED/CANCELLED)
   - Botão "Nova Transferência" (OPERATOR+)
   - Modal criação: produto, lote, quantidade, localização origem, destino
   - Botão Confirmar → PUT /transfers/{id}/confirm
   - Botão Cancelar → PUT /transfers/{id}/cancel

4. **React Query hooks** — `lib/api/audit.ts` + `lib/api/transfers.ts`:
   - `useAuditLog(filters)` — GET /audit/log paginado, staleTime 60s
   - `useRetentionConfig()` — GET /audit/retention-config
   - `useUpdateRetentionConfig` — PUT mutation
   - `useTransfers(filters)` — GET /transfers paginado, staleTime 30s
   - `useCreateTransfer` — POST mutation
   - `useConfirmTransfer(id)` — PUT mutation
   - `useCancelTransfer(id)` — PUT mutation

5. **TypeScript types** — `types/audit.ts` + `types/transfers.ts`

6. **Proteção por role**: OPERATOR+ para ver/criar; ADMIN para export e retenção

7. **Testes unitários** (≥ 8 novos)
