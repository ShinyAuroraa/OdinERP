# Story 16.3: GitHub Actions CD Pipeline

## Status

Done

## Story

**As a** engenheiro DevOps,
**I want** um pipeline de CD que builde a imagem Docker e a publique no GitHub Container Registry,
**so that** o deploy para o cluster K8s seja automático após CI verde em `main`.

## Complexity

**Estimativa:** M (Medium) · ~0.5 dia

## Dependencies

- Story 16.1 ✅ (Dockerfile multi-stage)
- Story 16.2 ✅ (CI Pipeline passando)

## Out of Scope

- ❌ K8s manifests (Story 16.4)
- ❌ Health endpoints (Story 16.5)
- ❌ TLS/cert-manager em produção

## Acceptance Criteria

1. **`.github/workflows/cd.yml`** disparado após `CI — WMS Web` completar com sucesso em `main`
2. **Login** no GitHub Container Registry via `GITHUB_TOKEN` (sem secrets extras)
3. **Build + Push** da imagem com 2 tags: SHA curto + `latest`
4. **Registry:** `ghcr.io/{owner}/odin-wms-web:{sha}` e `:latest`
5. **Build args** (NEXT_PUBLIC_*) via `vars.*` do repositório (GitHub Variables)
6. **Cache Docker** via GitHub Actions cache (mode=max)
7. **Deploy K8s** condicional: só executa se secret `KUBECONFIG` estiver configurado
