# Story 5.2: Packing — Estação de Embalagem

## Status

Draft

## Story

**As a** operador de armazém (WMS Operator),
**I want** processar as ordens de picking concluídas numa estação de packing — verificando itens por scanner, registrando peso e embalagem, gerando etiqueta GS1-128 com SSCC e confirmando o fechamento da caixa —
**so that** cada volume expedido seja verificado, identificado com código único de rastreamento (SSCC) e documentado, garantindo a integridade do envio e a continuidade do fluxo outbound até a expedição.

### AC6 — `PackingService.scanItem()`: verificação por código de barras

**Given** uma `PackingOrder` com status `IN_PROGRESS` e um `PackingItem`,
**When** `scanItem(tenantId, packingOrderId, packingItemId, request)` é chamado,
**Then**:
- Valida que o `request.barcode` corresponde ao produto do `PackingItem`:
  - Busca `ProductWms` pelo `packingItem.productId`
  - Barcode aceito se corresponde ao `product.sku` (case-insensitive) **ou** ao `lot.lotNumber` (se item tem lote) (case-insensitive)
  - Se barcode não corresponde a nenhum → `BusinessException` ("Barcode não corresponde ao produto esperado. Esperado: {sku}") → HTTP 400
- Marca `scanned = true`, `scanned_at = now()`, `scanned_by = request.scannedBy`
- Retorna `PackingItemResponse` atualizado
- **Regras de negócio:**
  - Item já escaneado → `BusinessException` ("Item já foi escaneado") → HTTP 409
  - `PackingOrder` não `IN_PROGRESS` → `BusinessException` → HTTP 409
  - Tenant isolation: 403 para tenantId divergente

### AC7 — `PackingService.setPackageDetails()`: peso, embalagem e dimensões

### AC8 — `PackingService.generateLabel()`: SSCC + GS1-128

**Given** uma `PackingOrder` com status `IN_PROGRESS`,
**When** `generateLabel(tenantId, packingOrderId)` é chamado,
**Then**:
- Gera SSCC de 18 dígitos caso `sscc` ainda seja `null`:
  - Algoritmo: `[ExtensionDigit(1=0)] + [TenantHash-7d] + [SerialMillis-9d] + [GS1CheckDigit(1)]`
  - `tenantHash = abs(tenantId.getMostSignificantBits()) % 10_000_000L` → 7 dígitos com zero-padding
  - `serialMillis = System.currentTimeMillis() % 1_000_000_000L` → 9 dígitos com zero-padding
  - `checkDigit` via GS1 Modulo-10 (multiplicadores alternados 3-1 da direita para a esquerda, `(10 - sum % 10) % 10`)
  - Em caso de colisão no `UNIQUE(sscc)`: gera novo SSCC com `serialMillis + 1` (max 3 tentativas)
- Gera barcode GS1-128 como Base64 PNG:
  - Usando ZXing `Code128Writer`: `writer.encode("(00)" + sscc, BarcodeFormat.CODE_128, 400, 150)`
  - Converte `BitMatrix` para PNG → Base64
- Armazena o `sscc` gerado em `packing_orders.sscc`
- Retorna `PackingLabelResponse { sscc, barcodeBase64, barcodeFormat: "GS1_128" }`
- Se `sscc` já gerado (não nulo) → retorna `PackingLabelResponse` com o SSCC existente + novo barcode Base64 (idempotente)

### AC9 — `PackingService.completePacking()`: fechamento e evento Kafka

**Given** uma `PackingOrder` com status `IN_PROGRESS`,

- Retorna `PackingOrderResponse` completo (com itens)

**Geração de SSCC (GS1 Modulo-10):**
```java
private String generateSSCC(UUID tenantId) {
    long tenantHash = Math.abs(tenantId.getMostSignificantBits()) % 10_000_000L;
    long serial = System.currentTimeMillis() % 1_000_000_000L;
    // payload = "0" + 7d(tenantHash) + 9d(serial) → 17 chars
    String payload = String.format("0%07d%09d", tenantHash, serial);
    int checkDigit = calculateGS1CheckDigit(payload);
    return payload + checkDigit; // 18 chars
}

private int calculateGS1CheckDigit(String payload17) {
    // GS1 Modulo-10: multiply alternating 3-1 from right, sum, (10 - sum%10)%10
    int sum = 0;
    for (int i = 0; i < 17; i++) {
        int digit = payload17.charAt(16 - i) - '0';
        sum += (i % 2 == 0) ? digit * 3 : digit;
    }
    return (10 - (sum % 10)) % 10;
}
```

**CodeRabbit Focus Areas**:
