## Story

**As a** usuário,
**I want** fazer login com minhas credenciais corporativas via Keycloak,
**so that** eu acesse o ODIN CRM com meu perfil de acesso correto sem criar uma nova senha.

## Acceptance Criteria

1. Fluxo OIDC via Keycloak implementado: redirect → callback → sessão estabelecida (`lib/auth.ts` com NextAuth 5 + Keycloak provider; `app/api/auth/[...nextauth]/route.ts` exportando handlers)
2. `middleware.ts` (Edge Runtime) protege rotas do dashboard: ausência de sessão válida redireciona para `/login`
3. Hook `useCurrentUser()` em `lib/hooks/use-current-user.ts` retorna `{ name, email, roles }` extraídos da sessão; `SessionProvider` disponível via `components/layout/Providers.tsx` montado em `app/layout.tsx`
4. Logout encerra sessão Next.js e redireciona para `end_session_endpoint` do Keycloak (SSO global logout) — via `signOut()` com parâmetro Keycloak-specific
5. JWT callback em `lib/auth.ts` realiza silent refresh: armazena `refresh_token` e `expires_at`; renova `access_token` automaticamente antes de expirar via endpoint `{KEYCLOAK_ISSUER}/protocol/openid-connect/token`
6. Página `/login` exibe mensagem de erro em pt-BR mapeada a partir do searchParam `?error=` do NextAuth (ex: `OAuthSignin`, `OAuthCallback`, `AccessDenied`)

## Scope

### IN (esta story)
- Fluxo OIDC completo: login → callback → sessão → logout
- Proteção de rotas `/dashboard/**` via `middleware.ts`
- Hook `useCurrentUser()` expondo name, email, roles
- Silent refresh automático de `access_token`
- Mensagem de erro na tela de login

### OUT (fora desta story)
- Registro/cadastro de novos usuários (gerenciado no Keycloak Admin — Story 1.7)

- Integração com `GET /api/v1/me` do backend (Story 1.5 usa dados da sessão NextAuth; a chamada ao backend virá com o Axios client — Story 2.x)
- Tela de perfil do usuário (Story 1.7)

    ```typescript
    import NextAuth from 'next-auth';
    import Keycloak from 'next-auth/providers/keycloak';
    // NÃO importar lib/env aqui — middleware.ts (Edge Runtime) importa este arquivo

    export const { handlers, auth, signIn, signOut } = NextAuth({
      providers: [
        Keycloak({
          clientId: process.env.KEYCLOAK_CLIENT_ID!,
          clientSecret: process.env.KEYCLOAK_CLIENT_SECRET!,
          issuer: process.env.KEYCLOAK_ISSUER!,
        }),
      ],
      callbacks: {
        async jwt({ token, account }) {
          if (account) {
            return {
              ...token,
              accessToken: account.access_token,
              refreshToken: account.refresh_token,
              expiresAt: account.expires_at,
            };
          }
          // Token ainda válido
          if (Date.now() < (token.expiresAt as number) * 1000) return token;
          // Expirado — realizar silent refresh
          return await refreshAccessToken(token);
        },
        async session({ session, token }) {
          session.accessToken = token.accessToken as string;
          session.user.roles = (token.roles as string[]) ?? [];
          return session;
        },
      },
    });
    ```

### Silent Refresh — Implementação
[Source: architecture.md#Section-7.1]

Keycloak emite `refresh_token` no token exchange. NextAuth v5 armazena tokens no JWT (cookie httpOnly). O callback `jwt` é invocado a cada requisição para verificar validade. Se `expires_at < now()`, chama o endpoint token do Keycloak com `grant_type=refresh_token`. Em falha de refresh: session expira e usuário é redirecionado para login.

### Logout SSO
[Source: architecture.md#Section-7.1]

NextAuth v5 + Keycloak provider detecta `end_session_endpoint` via OIDC discovery (`{KEYCLOAK_ISSUER}/.well-known/openid-configuration`). `signOut({ callbackUrl: '/login' })` apaga a sessão Next.js **e** redireciona para o end_session_endpoint do Keycloak, encerrando a sessão global no SSO.

### Riscos Conhecidos

| Risco | Mitigação |
|---|---|
| `lib/auth.ts` importa `lib/env.ts` (server-only) → crash no Edge Runtime do `middleware.ts` | **RESOLVIDO em Task 2.2**: usar `process.env.X!` diretamente em `lib/auth.ts`, não importar `lib/env` |
| NextAuth v5 ainda em desenvolvimento ativo — API pode mudar entre minor versions | Fixar versão exata no `package.json` após instalação; verificar changelog antes de upgrade |
| Keycloak `end_session_endpoint` requer `id_token_hint` para logout global em versões 24+ | Se logout não encerrar sessão SSO, adicionar `id_token_hint: token.idToken` no callback de logout do NextAuth |
| `KEYCLOAK_CLIENT_SECRET=local-secret` no `.env.local` pode não corresponder ao secret real do Keycloak local | Verificar no Keycloak Admin Console o secret do client `odin-crm-frontend` antes de testar |
| Silent refresh com Keycloak: `refresh_token` pode expirar (Keycloak tem `Refresh Token Max Reuse` e `SSO Session Idle`) | Se refresh_token expirado → redirecionar para `/login`; documentar no `RefreshAccessTokenError` handler |
